# Security Policy

## Supported Versions

Only the latest release is actively supported with security updates.

| Version | Supported          |
| ------- | ------------------ |
| Latest  | :white_check_mark: |
| Older   | :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability in Days Left, please report it responsibly.

**Do not open a public issue.**

Instead, contact the maintainer directly by opening a private security advisory on GitHub or emailing the project maintainer.

We will:

- Acknowledge receipt within 48 hours
- Provide an initial assessment within 5 days
- Work on a fix and release timeline
- Credit you in the release notes (unless you prefer anonymity)

## Security Considerations

Days Left is a local macOS utility with the following security characteristics:

- No network access
- No data collection or telemetry
- No external dependencies
- Runs only in the menu bar with minimal privileges

## Best Practices for Users

- Download only from official releases
- Verify the DMG checksum when available
- Report any suspicious behavior immediately
